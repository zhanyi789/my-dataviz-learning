# Workflow Map

## How Components Connect
1. `learning/` defines weekly goals and exercises.
2. `context/` provides reusable rules and frameworks.
3. `prompts/` turns context into repeatable AI requests.
4. `agents/` applies role-based feedback or guidance.
5. `datasets/` stores data and documentation inputs.
6. `projects/` contains student outputs and reflections.
7. `examples/` supports comparison and critique practice.

## Suggested Loop
Learn -> Prompt -> Generate -> Critique -> Revise -> Document.
