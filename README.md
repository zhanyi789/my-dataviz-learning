# Generative AI Applications in Data Visualization

## What This Project Is For
This repository is a course workspace for teaching and learning practical uses of generative AI in data visualization. It is designed for students, instructors, and AI tools to collaborate using shared, well-structured context.

## Folder Structure
- `syllabus/`: Course-level overview and teaching direction.
- `learning/weekXX/`: Weekly topic notes, self-study guides, and short quizzes.
- `context/`: Reusable context files for better AI prompting and consistent outputs.
- `agents/`: Reusable AI role definitions (critic, designer, coach).
- `prompts/`: Prompt templates with placeholders for repeatable workflows.
- `skills/`: Reusable skill summaries students build over time.
- `datasets/`: Raw data, processed data, and codebooks.
- `projects/`: Student project folders for staged work.
- `examples/`: Good and bad visualization examples for critique practice.
- `tools/`: Small starter templates for R, Python, and Colab workflows.
- `docs/`: Cross-cutting documentation and conventions.

## How Students Can Use AI Here
1. Start in `learning/weekXX/topic.md` to understand the weekly focus.
2. Use `context/` files as shared prompt context.
3. Copy templates from `prompts/` and fill placeholders.
4. Invoke role files in `agents/` to get targeted feedback.
5. Store data notes in `datasets/codebooks/` before charting.
6. Build outputs in `projects/` and compare against `examples/`.

## How Instructors Can Expand This Project
- Add more weekly folders (`week04`, `week05`, etc.) with the same pattern.
- Extend `agents/` with assignment-specific reviewer roles.
- Add domain-specific prompt templates in `prompts/`.
- Add rubrics, grading guides, and sample solutions in `docs/`.
- Add curated datasets and updated codebooks in `datasets/`.

## Recommended Workflow
1. Read weekly `topic.md`.
2. Complete `study.md` using `context/` + `prompts/`.
3. Build one chart prototype with a `tools/` starter file.
4. Run critique with `agents/dataviz_critic.md`.
5. Revise and document decisions in your project `README.md`.
6. Finish with `quiz.md` and a short reflection.
